Q-1: What is the difference between var, let, and const?

Ans:

i) var - It can be re-declared and update the value of a variable, and it hoists the data to the top.

ii) let - It gives the scope to the user to update a variable value when needed, but it cannot be re-declared.

iii) const - It is fixed. The user can't update the value of a variable, and it cannot be re-declared.


Q-2: What is the spread operator (...)?

Ans:

The spread operator (...) is used to create a copy of reference-type data such as arrays and objects. Normally, when we modify an array or object, the original data may also be affected. To avoid this, we use the spread operator to make a separate copy first. After copying, we can add or change values in the new array or object without changing the original one.


Q-3: What is the difference between map(), filter(), and forEach()?

Ans:

i) map() – The map() method goes through each element of an array, applies a function to it, and creates a new array with the modified values.

ii) filter() – The filter() method checks every element of an array and returns a new array containing only the elements that meet a specific condition.

iii) forEach() – The forEach() method also iterates over each element of an array and executes a function, but it does not return a new array.


Q-4: What is an arrow function?

Ans:

An arrow function (=>) is a concise syntax introduced in ES6 that allows developers to write functions in a shorter and simpler way compared to traditional function expressions.


Q-5: What are template literals?

Ans:

Template literals ( ) are a feature in JavaScript that use backticks instead of single (' ') or double (" ") quotes. They make it easier to write multi-line strings and allow variables or dynamic values to be inserted directly into the text.
